# 网络小说阅读管理工具 - 需求分析文档

## 一、项目概述

### 1.1 项目名称
iBook - 个人网络小说阅读管理工具

### 1.2 项目背景
针对网络小说阅读爱好者，提供一个本地化的小说管理工具，帮助用户记录、管理和追踪自己的阅读历程，同时支持本地文档管理和字数统计功能。

### 1.3 目标用户
- 网络小说阅读爱好者
- 需要管理大量小说阅读记录的用户
- 希望本地化管理小说文档的用户

### 1.4 核心价值
- 集中化管理阅读记录
- 自动化信息检索，减少手动录入
- 本地文档管理，离线阅读支持
- 阅读数据统计与分析

---

## 二、功能需求

### 2.1 核心功能

#### 2.1.1 小说信息管理

**功能描述：** 记录和管理网络小说的基本信息

**数据字段：**
- 书名（必填）
- 作者（建议新增）
- 字数（必填）
- 封面图片
- 所在平台（如：起点中文网、晋江文学城等）
- 类型/标签（如：玄幻、都市、言情等）
- 描述/简介
- ISBN/书籍唯一标识（可选）
- 添加时间（系统自动）
- 更新时间（系统自动）

**操作功能：**
- 新增小说记录
- 编辑小说信息
- 删除小说记录
- [已完成] 批量导入（已实现，详见 2.2.6 批量操作）
- [已废弃] 批量导出（已废弃）

#### 2.1.2 小说信息检索

**功能描述：** 从 https://www.youshu.me/ 平台自动检索小说信息

**检索方式：**
- 按书名检索
- 按作者检索
- 按书名+作者组合检索

**检索结果处理：**
- 展示检索到的多个结果供用户选择
- 支持预览检索到的详细信息
- 一键导入选中的小说信息
- 检索失败时提示用户手动录入

**优化建议：**
- 支持模糊匹配
- 缓存检索结果，避免重复请求
- 检索历史记录

#### 2.1.3 手动录入

**功能描述：** 当检索不到或用户希望自定义时，支持手动录入所有信息

**录入界面：**
- 表单式录入界面
- 字段验证（必填项提示）
- 封面图片上传（支持拖拽）
- 实时保存草稿

#### 2.1.4 文档管理

**功能描述：** 上传并管理小说对应的电子文档

**支持的文档格式：**
- TXT（纯文本）
- EPUB（电子书标准格式）
- MOBI（Kindle格式）
- PDF
- AZW/AZW3
- DOC/DOCX（Word文档）

**文档操作：**
- 上传文档（支持拖拽）
- 删除文档
- 替换文档
- 下载/导出文档
- 在系统中打开文档
- 文档预览（可选）

**存储方式：**
- 本地文件系统存储
- 文件路径记录在数据库
- 文件重命名规则（避免冲突）

#### 2.1.5 字数统计

**功能描述：** 自动计算上传文档的字数，并提供字数数据源选择

**字数计算规则：**
- TXT：统计纯文本字符数（排除空格、换行符）
- EPUB/MOBI/AZW：解析电子书内容后统计
- PDF：提取文本后统计
- DOC/DOCX：提取文本内容后统计

**字数数据源管理：**
- 检索来源字数
- 文档计算字数
- 手动输入字数
- 用户可选择以哪个为准（设置主字数源）
- 显示所有字数源及差异

**字数差异提醒：**
- 当多个来源字数差异超过一定比例时（如10%），给出提示

## 三、非功能需求

### 3.1 性能要求

- 应用启动时间 < 3秒
- 小说列表加载时间 < 1秒（1000条记录内）
- 文档上传响应及时（显示进度条）
- 检索响应时间 < 5秒
- 支持10,000+小说记录

### 3.2 可用性要求

- 界面简洁直观，符合Windows UI规范
- 支持快捷键操作
- 错误提示友好明确
- 操作可撤销（如删除前确认）

### 3.3 兼容性要求

- 支持Windows 10及以上版本
- 分辨率适配：1920x1080、1366x768等主流分辨率
- 支持高DPI显示

### 3.4 安全性要求

- 本地数据加密存储（可选）
- 备份数据完整性校验
- 防止数据丢失（操作日志、定期备份）

### 3.5 可维护性要求

- 代码模块化，易于扩展
- 日志记录（错误日志、操作日志）
- 版本更新机制

## 四、技术架构建议

### 4.1 技术栈选型

#### 方案一：Electron + Vue/React（推荐）
**优点：**
- 跨平台（未来可扩展到macOS、Linux）
- 前端技术栈，开发效率高
- 丰富的UI组件库
- 易于实现现代化界面

**缺点：**
- 应用体积较大
- 内存占用较高

**适用场景：** 希望快速开发，界面美观，未来考虑跨平台

#### 方案二：C# + WPF
**优点：**
- 原生Windows应用，性能好
- 资源占用低
- 与Windows系统集成度高
- XAML设计界面直观

**缺点：**
- 仅限Windows平台
- 学习曲线相对陡峭

**适用场景：** 专注Windows平台，追求性能和系统集成

#### 方案三：Python + PyQt/PySide
**优点：**
- Python生态丰富，文档处理库多
- 开发速度快
- 跨平台

**缺点：**
- 打包后体积较大
- 性能不如原生应用

**适用场景：** 快速原型开发，文档处理需求多

### 4.2 数据存储方案

#### 本地数据库：SQLite
**理由：**
- 轻量级，无需安装
- 支持SQL查询
- 数据结构化存储
- 易于备份（单文件）

**数据表设计草案：**
- `books` 表：小说基本信息
- `documents` 表：文档信息
- `reading_progress` 表：阅读进度
- `notes` 表：笔记书评
- `tags` 表：标签
- `book_tags` 表：书籍标签关联

### 4.3 第三方库/服务

- **文档解析：**
  - epub: epub.js / epublib
  - pdf: PDF.js / PyPDF2
  - docx: python-docx / DocumentFormat.OpenXml
  
- **网络请求：** Axios / requests

- **UI组件库：** 
  - Element Plus / Ant Design (Electron)
  - Material Design (WPF)
  
- **图表库：** ECharts / Chart.js

## 五、数据模型设计

### 5.1 小说信息表 (books)

```sql
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL,           -- 书名
    author VARCHAR(100),                   -- 作者
    cover_url VARCHAR(500),                -- 封面URL或本地路径
    platform VARCHAR(100),                 -- 所在平台
    category VARCHAR(50),                  -- 类型
    description TEXT,                      -- 描述
    word_count_source VARCHAR(20),         -- 字数来源: 'search'/'document'/'manual'
    word_count_search INTEGER,             -- 检索到的字数
    word_count_document INTEGER,           -- 文档计算字数
    word_count_manual INTEGER,             -- 手动输入字数
    word_count_display INTEGER,            -- 显示字数（用户选择的主字数）
    isbn VARCHAR(50),                      -- ISBN
    source_url VARCHAR(500),               -- 来源URL
    reading_status VARCHAR(20) DEFAULT 'unread',  -- 阅读状态
    personal_rating DECIMAL(2,1),          -- 个人评分
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 文档表 (documents)

```sql
CREATE TABLE documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    book_id INTEGER NOT NULL,              -- 关联书籍ID
    file_name VARCHAR(200) NOT NULL,       -- 文件名
    file_path VARCHAR(500) NOT NULL,       -- 文件路径
    file_type VARCHAR(20) NOT NULL,        -- 文件类型
    file_size INTEGER,                     -- 文件大小(bytes)
    word_count INTEGER,                    -- 计算出的字数
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
```

### 5.3 阅读进度表 (reading_progress)

```sql
CREATE TABLE reading_progress (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    book_id INTEGER NOT NULL,
    current_chapter VARCHAR(200),          -- 当前章节
    current_page INTEGER,                  -- 当前页数
    progress_percentage DECIMAL(5,2),      -- 阅读百分比
    last_read_at DATETIME,                 -- 最后阅读时间
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
```

### 5.4 笔记表 (notes)

```sql
CREATE TABLE notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    book_id INTEGER NOT NULL,
    note_type VARCHAR(20),                 -- 类型: 'review'/'note'/'excerpt'
    content TEXT,                          -- 内容
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
```

### 5.5 标签表 (tags)

```sql
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    tag_name VARCHAR(50) UNIQUE NOT NULL,
    color VARCHAR(20),                     -- 标签颜色
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 5.6 书籍标签关联表 (book_tags)

```sql
CREATE TABLE book_tags (
    book_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY (book_id, tag_id),
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

## 八、风险与挑战

### 8.1 技术风险

1. **网页爬虫稳定性**
   - 风险：youshu.me 网站结构变化导致爬虫失效
   - 应对：设计灵活的爬虫架构，支持快速调整；提供手动录入兜底

2. **多格式文档解析**
   - 风险：部分电子书格式复杂，字数统计不准确
   - 应对：使用成熟的解析库，提供手动校正功能

3. **性能问题**
   - 风险：数据量大时，查询和渲染性能下降
   - 应对：分页加载、虚拟滚动、索引优化

### 8.2 产品风险

1. **功能过载**
   - 风险：功能太多导致界面复杂，用户学习成本高
   - 应对：分阶段开发，先做MVP，逐步添加功能

2. **用户数据安全**
   - 风险：数据丢失或损坏
   - 应对：定期自动备份，提供数据导出功能

## 九、补充建议与考虑点

### 9.1 未考虑到的边界场景

1. **重复添加处理**
   - 当用户尝试添加已存在的书籍时，如何处理？
   - 建议：检测重复（基于书名+作者），提示用户是否覆盖或忽略

2. **网络请求超时**
   - 检索时网络不佳或服务器无响应
   - 建议：设置合理的超时时间，提供重试机制

3. **文档文件损坏**
   - 上传的文档文件本身已损坏，无法解析
   - 建议：文件上传时进行完整性校验，解析失败时给出明确提示

4. **大文件上传**
   - 小说文档可能很大（如100MB+的PDF）
   - 建议：限制文件大小或实现分块上传，显示进度条

5. **数据迁移**
   - 版本升级时，数据库结构可能变化
   - 建议：设计数据库版本管理机制，支持平滑迁移

6. **封面图片失效**
   - 如果封面是网络URL，可能失效
   - 建议：下载封面到本地，或提供默认封面

### 9.2 用户体验优化建议

1. **智能推荐**
   - 基于用户的阅读历史和评分，推荐类似的小说
   - 可以对接推荐API或本地简单算法

2. **快捷操作**
   - 右键菜单快捷操作（删除、编辑、打开文档）
   - 键盘快捷键（Ctrl+N新建、Ctrl+F搜索等）

3. **拖拽支持**
   - 支持直接拖拽文档文件到窗口进行添加
   - 支持拖拽图片更新封面

4. **暗色模式**
   - 提供护眼的暗色主题
   - 根据系统主题自动切换

5. **全局搜索**
   - 支持全局快捷键唤醒搜索（如Alt+Space）
   - 搜索结果高亮

### 9.3 未来扩展方向

1. **社交功能**
   - 分享阅读列表给朋友
   - 导出阅读报告为图片分享到社交媒体

2. **移动端App**
   - 开发配套的移动端App，数据云同步
   - 在手机上也能查看和管理

3. **阅读器集成**
   - 内置简单的阅读器，直接在应用内阅读
   - 集成TTS（文本转语音）功能

4. **作者跟踪**
   - 关注喜欢的作者，自动检索其新作品

5. **书单功能**
   - 创建主题书单（如"2024年最佳"）
   - 导出书单为Markdown或HTML

## 十、总结

本需求分析文档从核心功能出发，逐步扩展到高级功能和优化方向。建议采用**迭代式开发**，先实现MVP（最小可行产品），包含基本的小说信息管理、检索、文档上传和字数统计功能，快速验证产品价值后，再逐步添加阅读进度、统计分析等高级功能。

**核心优势：**
- 本地化管理，数据安全可控
- 自动化检索，减少手动录入
- 多维度统计，了解阅读习惯
- 功能扩展性强，可持续迭代
